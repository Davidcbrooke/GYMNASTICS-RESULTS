Remote Judging — Setup Guide

Everything needed to set up and run the NGL Remote Judging system: the physical kit, the software on the competition laptop, and the routine for competition day. Written for the person running the venue — no technical background assumed.

Never put passwords in this page. This guide is publicly readable. Wherever it says YOUR-CAMERA-PASSWORD or similar, use the real value from your private notes — do not edit real passwords into this file.

1 · How the system fits together

Cameras at each apparatus plug into a PoE switch, which powers them and carries their video. The switch connects to our own travel router, which connects to whatever internet the venue provides. The competition laptop, also wired to the switch, runs two programs:

  • MediaMTX — pulls video from the cameras and converts it into a format web browsers can play.
  • cloudflared — an encrypted tunnel that carries that video out to feed.ngljudge.com.

Remote judges use the Remote Judging Control Centre web page. It shows the live video, the Stop/Go rotation controller, the rules reference, and Elevien scoring — all in one place. Two separate logins protect it: the page itself (email + password) and the video feed (email + one-time PIN, approved addresses only).

3 · Physical wiring

  1. Each camera → Cat6 cable → any numbered PoE port on the switch. The camera powers itself from the cable; allow 30–60 seconds for it to boot.
  2. Laptop → USB-Ethernet adapter → Cat6 → any spare PoE port on the switch.
  3. Switch uplink port → Cat6 → the Beryl router's LAN port.
  4. Beryl's WAN port → Cat6 → the venue's internet point (wall port or their router's LAN port).
  5. Power the Beryl with its USB-C adapter (needs a 20W+ adapter).
Shared buildings: never plug the switch's uplink directly into an unknown wall port or someone else's router. Always go through our own Beryl router — it keeps the cameras invisible to everyone else on the building's network.

4 · Router setup (GL.iNet Beryl)

Already configured — normally nothing to do beyond plugging it in. For reference, first-time setup was:

  1. Connect a laptop/phone to the Beryl's own wifi (name and password printed on its underside).
  2. A setup page opens automatically (or go to 192.168.8.1). Set the admin password; defaults are fine for everything else.

The Beryl gives all our devices addresses starting 192.168.8.x. To see what's connected (including camera IP addresses): browse to 192.168.8.1 → Clients.

5 · Camera setup (Reolink)

For each new camera, one-time setup:

  1. Install the Reolink Client (Windows desktop app from reolink.com) on the laptop.
  2. Plug the camera into the PoE switch. The client scans the network and finds it; add it and set its admin password (same password for every camera, kept in your private notes).
  3. In the camera's settings: Network → Advanced → Server Settings → tick RTSP → Save. Video cannot leave the camera without this.
  4. Note the camera's IP address (from the Beryl's Clients page, or the Reolink Client).
Quick test: in VLC (Media → Open Network Stream) paste
rtsp://admin:YOUR-CAMERA-PASSWORD@CAMERA-IP:554/h264Preview_01_sub
Live video = camera fully working.

6 · Laptop preparation

  • Wire it to the switch via the USB-Ethernet adapter. Turn the laptop's wifi off so all traffic goes over the wire.
  • Quit NordVPN (or any VPN) — it breaks the connection between the laptop, cameras, and tunnel.
  • Windows Settings → System → Power: set the laptop to never sleep when plugged in. A sleeping laptop kills the video for every judge.
  • Keep it on mains power for the whole competition.

7 · MediaMTX (video server)

MediaMTX lives in a folder on the laptop (currently C:\Users\info\Downloads\mediamtx_v1.19.2_windows_amd64) containing mediamtx.exe and its settings file mediamtx.yml.

The settings file lists each camera. Example:

paths:
  camera1:
    source: rtsp://admin:YOUR-CAMERA-PASSWORD@192.168.8.200:554/h264Preview_01_sub
    sourceOnDemand: no

  camera2:
    source: rtsp://admin:YOUR-CAMERA-PASSWORD@192.168.8.222:554/h264Preview_01_sub
    sourceOnDemand: no

To run it: double-click mediamtx.exe. A black window opens and must stay open. Within a few seconds the log should mention each camera as ready.

Checkpoint: on the laptop, browse to http://localhost:8889/camera1 — live video in the browser means MediaMTX is working.

8 · Cloudflare tunnel

The tunnel (named ngl-feed) is already created and linked to feed.ngljudge.com. Day to day, it only needs to be run:

  1. Open Command Prompt (Windows key → type cmd → Enter).
  2. Paste and run:
cloudflared tunnel --config C:\Users\info\Downloads\mediamtx_v1.19.2_windows_amd64\tunnel-config.yml run

A second black window opens showing "Registered tunnel connection" lines. It must also stay open.

Checkpoint: on a phone using mobile data, go to https://feed.ngljudge.com/camera1. Expect the email/PIN login, then live video.

If the laptop is ever replaced

The tunnel must be re-authorised on the new machine: install cloudflared, then run cloudflared tunnel login (sign into the NGL Cloudflare account, pick ngljudge.com), cloudflared tunnel create ngl-feed2, cloudflared tunnel route dns ngl-feed2 feed.ngljudge.com (deleting the old DNS record first), and update tunnel-config.yml with the new tunnel name and credentials file path it prints.

9 · Competition-day startup checklist

  1. Wire everything per section 3. Power the switch, router, and cameras. Wait one minute.
  2. Laptop on, mains power, wifi off, VPN off.
  3. Double-click mediamtx.exe → window stays open.
  4. Run the tunnel command (section 8) → second window stays open.
  5. Phone check on mobile data: feed.ngljudge.com/camera1 shows video after PIN login.
  6. Open the Remote Judging Control Centre page, sign in, open an apparatus, confirm video appears (use Authorise video access → PIN → Reload feed if prompted).
  7. Set each apparatus display device (laptop/screen at the venue) to its full-screen display link and leave it running.
Both black windows must stay open all day. Closing either one cuts video for every judge. Minimise them, don't close them.

10 · Adding and removing judges

Each judge needs to be added in two places, using the same email address in both:

Control Centre login (Firebase)

  1. console.firebase.google.com → project saadi-rotation → Authentication → Users → Add user.
  2. Enter the judge's email and set them a password. Tell them the password yourself — no automatic email is sent.
  3. To remove a judge: same screen → three-dot menu on their row → Delete.

Video feed access (Cloudflare)

  1. Cloudflare dashboard → Zero Trust → Access controls → Applications → feed → its policy (NGL Judges Only).
  2. Add the judge's email to the Include → Emails list (press Enter so it becomes a confirmed tag) → Save.
  3. To remove: delete their email from the same list. Their access ends when their current session (max 8 hours) expires.

11 · Adding another camera

  1. Physically install and set up the camera (sections 3 and 5). Note its new IP address.
  2. Edit mediamtx.yml: copy an existing camera block, give it the next name (e.g. camera3), and set its IP.
  3. Restart MediaMTX (close its window, double-click mediamtx.exe again).
  4. Edit the Control Centre page file: find the CAMERA_FEEDS map near the top of the script and set the apparatus to the new camera name, e.g. beam: 'camera3'. Re-upload the file to GitHub.

12 · Troubleshooting

SymptomLikely cause and fix
Judge page video box says "refused to connect"The judge hasn't signed in for video this session. Click Authorise video access, complete the email/PIN, then Reload feed.
"Bad gateway" / error 502 at feed.ngljudge.comThe tunnel is running but MediaMTX isn't (or vice versa). Check both black windows are open on the laptop; restart whichever is missing.
feed.ngljudge.com won't load at allThe tunnel window isn't running, or the laptop has no internet. Check the tunnel window and the Beryl's WAN connection.
MediaMTX log shows camera errorsCamera unplugged, wrong IP in mediamtx.yml (IPs can change if the router was reset — check the Beryl's Clients page), or RTSP got disabled on the camera.
Reolink Client finds no camerasCamera not powered (check switch port lights), or laptop on the wrong network — it must be wired to the switch (or on the Beryl's wifi), with VPN off.
VLC can't connect to a cameraRTSP not enabled and saved on that camera (section 5), wrong IP, or laptop on a different network to the cameras.
Everything worked, then all video died mid-sessionLaptop went to sleep, a window was closed, or VPN auto-started. Check power settings, both windows, and the system tray.
Judge can't log into the Control CentreCheck their account exists in Firebase Authentication → Users; reset the password from there if needed.
Judge gets "access denied" at video loginTheir email isn't on the Cloudflare policy list — section 10, second part.
